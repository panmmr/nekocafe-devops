# 实验三 完整操作手册

按照本手册操作，从零到全部完成，约需 2 小时。

---

## 准备：安装必要工具

### 步骤 0.1：安装 Docker Desktop
打开浏览器 → https://www.docker.com/products/docker-desktop/ → 下载 → 安装 → 重启电脑 → 打开 Docker Desktop。

验证：
```bash
docker --version
```
应该显示 `Docker version 2x.x.x`

### 步骤 0.2：安装 Python 依赖
```bash
pip install python-docx playwright
playwright install chromium
```

### 步骤 0.3：安装 Git（如果没有）
打开 https://git-scm.com/download/win → 下载 → 安装。

## 第一章：运行项目（30 分钟）

### 1.1 启动全部服务
打开 **PowerShell** 或 **终端**，执行：
```bash
cd "C:\Users\33492\Desktop\计算机23-2_230502209_潘美儒_实验三_v1.0\nekocafe-devops"
docker compose up -d --build
```
等待 2-3 分钟，看到 `Container nekocafe-reservation Started` 之类的提示就成功了。

### 1.2 验证服务正常
```bash
curl http://localhost:8000/health
```
应该返回：`{"status":"healthy","service":"reservation-service"}`

```bash
curl http://localhost:8001/health
```
应该返回：`{"status":"healthy","service":"member-service"}`

### 1.3 测试 API
打开浏览器 → 访问 http://localhost:8000/docs → 随便点一个接口 → 点 "Try it out" → 点 "Execute"

同样 http://localhost:8001/docs 也试一试。

## 第二章：GitHub 仓库（15 分钟）

### 2.1 初始化 Git
```bash
cd "C:\Users\33492\Desktop\计算机23-2_230502209_潘美儒_实验三_v1.0\nekocafe-devops"
git init
git add .
git commit -m "feat: NekoCafe DevOps pipeline — Experiment 3 complete"
```

### 2.2 创建 GitHub 仓库
1. 打开浏览器 → https://github.com → 登录你的账号
2. 点右上角 **+** → **New repository**
3. Repository name: `nekocafe-devops`
4. **选 Public**（公开）
5. **不要**勾选 "Add a README file"
6. 点 **Create repository**

### 2.3 推送代码
```bash
git remote add origin https://github.com/你的GitHub用户名/nekocafe-devops.git
git branch -M main
git push -u origin main
```
（替换"你的GitHub用户名"为实际用户名）

### 2.4 查看 CI 运行
推送后自动触发 CI。打开 https://github.com/你的GitHub用户名/nekocafe-devops/actions → 看到黄色圆点在转 → 等了 5-8 分钟 → 全部变绿色 ✅

## 第三章：截图（15 分钟）

### 3.1 Grafana 截图（自动）
先启动监控栈：
```bash
cd "C:\Users\33492\Desktop\计算机23-2_230502209_潘美儒_实验三_v1.0\nekocafe-devops"
docker compose -f docker-compose.yml -f docker-compose.monitoring.yml up -d
```
等待 1 分钟，然后运行：
```bash
python scripts/screenshot-grafana.py --username admin --password admin
```
截图自动保存到 `docs/screenshots/` 文件夹。

以后再次截图（免登录）：
```bash
python scripts/screenshot-grafana.py
```

### 3.2 GitHub Actions 截图（手动 2 张）
1. 浏览器打开 https://github.com/你的GitHub用户名/nekocafe-devops/actions
2. 点一次成功的 CI 运行
3. 看到所有 Job 绿色 ✅ 的页面 → **截图** → 保存为 `docs/screenshots/ci-success.png`
4. （可选失败截图）回到 Actions 列表 → 点一次失败的 → **截图** → `docs/screenshots/ci-failure.png`

也可以用命令行生成文字报告代替截图：
```bash
bash scripts/generate-ci-report.sh
```

## 附录：Minikube 本地模拟 K8s 部署（可选加分项）

```bash
# 安装 Minikube
winget install minikube

# 启动
minikube start --cpus=2 --memory=4096

# 部署 NekoCafé
helm install nekocafe ./k8s/helm/nekocafe \
  -f ./k8s/helm/nekocafe/values.yaml \
  -f ./k8s/helm/nekocafe/values-dev.yaml \
  --namespace nekocafe-dev --create-namespace

# 查看状态
kubectl get pods -n nekocafe-dev
minikube service nekocafe-reservation -n nekocafe-dev --url
```
